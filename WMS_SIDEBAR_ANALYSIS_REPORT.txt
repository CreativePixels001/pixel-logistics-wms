5. RECOMMENDED APPROACH TO CREATE COMMON/REUSABLE SIDEBAR
================================================================================

RECOMMENDED SOLUTION: JavaScript Component with Dynamic Injection
──────────────────────────────────────────────────────────────────

This approach maintains your current static HTML architecture while ensuring 
consistency and ease of maintenance.

┌─ STEP 1: Create Sidebar Component File ────────────────────────────────────┐
│                                                                             │
│ Create: frontend/WMS/js/sidebar-component.js                               │
│                                                                             │
│ This file will contain:                                                    │
│   • Complete sidebar HTML as a JavaScript template string                 │
│   • Function to detect current page and set active states                 │
│   • Function to inject sidebar into the DOM                               │
│   • Collapsible section state management                                  │
└─────────────────────────────────────────────────────────────────────────────┘

┌─ STEP 2: Modify HTML Files ────────────────────────────────────────────────┐
│                                                                             │
│ In each HTML file that needs the sidebar:                                 │
│                                                                             │
│ 1. REMOVE the entire <aside class="sidebar">...</aside> block              │
│                                                                             │
│ 2. ADD a placeholder div:                                                  │
│    <div id="wms-sidebar-mount"></div>                                      │
│                                                                             │
│ 3. INCLUDE the sidebar component script before closing </body>:           │
│    <script src="../js/sidebar-component.js"></script>                      │
│                                                                             │
│ 4. INITIALIZE the sidebar:                                                 │
│    <script>                                                                 │
│      document.addEventListener('DOMContentLoaded', function() {            │
│        initWMSSidebar('current-page-name');                                │
│      });                                                                    │
│    </script>                                                                │
└─────────────────────────────────────────────────────────────────────────────┘

┌─ STEP 3: Implementation Benefits ──────────────────────────────────────────┐
│                                                                             │
│ ✓ Single source of truth - Update sidebar once, applies everywhere        │
│ ✓ Automatic active state - Component detects current page                 │
│ ✓ Consistent navigation - All pages have identical menu structure         │
│ ✓ Easy maintenance - Add/remove menu items in one place                   │
│ ✓ No build tools required - Pure JavaScript solution                      │
│ ✓ Preserves static HTML - Works with current deployment                   │
│ ✓ Backward compatible - Can migrate pages incrementally                   │
└─────────────────────────────────────────────────────────────────────────────┘


ALTERNATIVE APPROACHES:
────────────────────────

APPROACH 2: Web Components (Modern browsers)
  Pros: Native browser support, fully encapsulated, reusable
  Cons: Requires modern browser, learning curve
  Implementation: <wms-sidebar active="dashboard"></wms-sidebar>

APPROACH 3: Server-Side Includes (Requires backend)
  Pros: Generated at build/serve time, SEO friendly
  Cons: Needs server infrastructure or build process
  Implementation: <?php include 'includes/sidebar.php'; ?>

APPROACH 4: Build-Time Templates (Using build tools)
  Pros: Best performance, full control, generates static HTML
  Cons: Requires build process (Gulp, Webpack, etc.)
  Implementation: {{> sidebar}} with Handlebars/EJS
